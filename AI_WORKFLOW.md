# AI Workflow Note

## AI Tools Used
I used AI assistance for scope planning, implementation guidance, debugging, documentation structure, and final checklist review.

## Where AI Helped
- Breaking the ambiguous prompt into a realistic MVP
- Choosing a safe feature order
- Speeding up code structure and test setup
- Drafting README, architecture, AI workflow, and submission notes

## What I Changed or Rejected
I did not blindly accept every AI suggestion. I rejected full authentication, real-time collaboration, complex database setup, advanced editor architecture, and enterprise access control because they were too risky for the timebox.

I chose seeded users, localStorage persistence, and simple sharing because they demonstrate the required workflow clearly.

## Verification
I manually tested the main user flow: create document, rename, edit, format, save, refresh, upload file, share document, and switch users.

I also ran npm test and npm run build. Both passed successfully.

## Reflection
I used AI as a planning and debugging assistant, but I verified the final product manually against the original requirement.
